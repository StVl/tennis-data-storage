# Проект: tennis-data — данные теннисного приложения

Ты — ассистент по обслуживанию данных мобильного теннисного приложения.
Приложение читает один файл `config.json` с GitHub Pages и просто его отрисовывает.
Само приложение ничего не обновляет. Актуальность данных обеспечивает этот репозиторий.

## Архитектура (важно понять до любых действий)

- `data/*.json` — ИСТОЧНИК ПРАВДЫ. Маленькие шарды, разбитые по частоте обновления.
  Именно их правят (ты или человек).
- `config.json` — СОБРАННЫЙ АРТЕФАКТ. Его читает приложение. Руками НЕ редактируется.
  Получается склейкой шардов скриптом `build_config.py`.
- `build_config.py` — детерминированная сборка без LLM: валидация шардов →
  пересчёт `playerCards` из матчей → склейка `config.json` → опционально git push.

Поток: правка шарда → `python3 build_config.py --check` → (по расписанию) сборка и пуш.

### Золотое правило
НИКОГДА не редактируй `config.json` напрямую и не пересчитывай `playerCards` руками —
это делает только `build_config.py`. Ты правишь ТОЛЬКО файлы в `data/`.

## Шарды

| Файл | Что | Источник | Частота |
|---|---|---|---|
| `data/matches_upcoming.json` | ближайшие матчи + `isLive`/`liveScore` | Sofascore | каждый час |
| `data/matches_past.json` | завершённые матчи | Sofascore | по факту завершения |
| `data/tournaments_upcoming.json` | ближайшие/идущие турниры | ATP | раз в день |
| `data/tournaments_past.json` | завершённые турниры | ATP | редко |
| `data/rankings.json` | рейтинг, очки, тренд (вынесено из профилей!) | ATP / Tennis Abstract | раз в неделю (пн) |
| `data/players.json` | профили игроков (без рейтинга и очков) | ATP | раз в неделю |
| `data/reference.json` | стили игры | — | почти никогда |

## Задачи обновления

Готовые промпты лежат в `prompts/`. Каждый работает ТОЛЬКО со своим шардом:
- `prompts/update_matches.md` — приоритет 1: ближайшие матчи и live-счёт (раз в час)
- `prompts/update_tournaments.md` — приоритет 2: ближайшие турниры (раз в день)
- `prompts/update_rankings.md` — приоритет 3: рейтинг и очки (по понедельникам)

Когда тебя просят «обнови матчи / турниры / рейтинг» — открой соответствующий промпт
и следуй ему дословно, затем прогони `python3 build_config.py --check`.

## Команды

```bash
python3 build_config.py            # собрать config.json
python3 build_config.py --check    # только валидация (после ручных правок шардов)
python3 build_config.py --push     # собрать и запушить, если изменилось
./run_task.sh prompts/update_matches.md   # выполнить задачу-обновление + сборка + пуш
```

## Настройка автозапуска (cron) — выполни по запросу пользователя

Если пользователь просит «настрой cron / автообновление»:
1. Узнай абсолютный путь к этой папке (`pwd`).
2. Убедись, что `run_task.sh` исполняемый: `chmod +x run_task.sh`.
3. Проверь, что в этой папке инициализирован git и настроен remote (`git remote -v`).
   Если нет — помоги настроить, без этого `--push` не сработает.
4. Покажи пользователю строки для `crontab -e` (подставив абсолютный путь):

```cron
0 * * * *   /АБС_ПУТЬ/run_task.sh prompts/update_matches.md      >> /tmp/tennis_matches.log 2>&1
30 7 * * *  /АБС_ПУТЬ/run_task.sh prompts/update_tournaments.md  >> /tmp/tennis_tourn.log   2>&1
0 8 * * 1   /АБС_ПУТЬ/run_task.sh prompts/update_rankings.md     >> /tmp/tennis_rank.log    2>&1
```

> macOS: если ноут часто спит, предложи launchd вместо cron (догоняет пропуски).
> Также cron-сессии не видят PATH из shell — в `run_task.sh` пути уже абсолютные,
> но проверь, что бинарь `claude` доступен (`which claude`), и при необходимости
> впиши его полный путь в `run_task.sh`.

## Принципы при правках данных
- Не выдумывай счёт, даты, составы — бери только с источника. Не уверен — оставь как было.
- Не меняй существующие `id` матчей/игроков/турниров.
- В матчах строго проверяется только `playerId` (должен быть в `players.json`);
  `opponentId` может быть `TBD` или игроком вне ростера — это нормально.
- JSON всегда валидный, UTF-8, отступ 2 пробела.
- В конце ВСЕГДА прогоняй `python3 build_config.py --check`.
