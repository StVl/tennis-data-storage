# Задача: обновить ближайшие матчи (приоритет 1, в т.ч. live раз в час)

Ты обновляешь данные теннисного приложения. Работаешь ТОЛЬКО с двумя файлами:
`data/matches_upcoming.json` и `data/matches_past.json`. Остальные шарды не трогаешь.

## Источник
Sofascore (расписание и счёт матчей ATP). Открой страницы ближайших матчей игроков,
которые есть в `data/players.json` (поле `id` — это playerId).

## Что сделать
1. Прочитай `data/matches_upcoming.json` и `data/players.json`.
2. Для каждого матча из `matches_upcoming`:
   - если матч ещё не начался — при необходимости поправь `startAt`, `stage`, `surface`,
     `tournament`, оставь `isLive: false`, `liveScore: null`, `status: "upcoming"`;
   - если матч идёт прямо сейчас — выстави `isLive: true` и обнови `liveScore`
     (строка вида "6-4, 3-2" — счёт по сетам/текущему гейму на момент проверки);
   - если матч завершился — поменяй `status` на `"completed"`, проставь `result`
     ("won"/"lost" с точки зрения `playerId`) и финальный `score`, убери поля
     `isLive`/`liveScore`, и ПЕРЕНЕСИ объект из `matches_upcoming.json`
     в `matches_past.json`.
3. Добавь новые ближайшие матчи игроков, которых ещё нет в `matches_upcoming`
   (id формируй по схеме существующих: `<playerId>_<турнир>_<стадия>_<год>`).

## Правила
- **НИКОГДА не помечай матч `completed` пока он не сыгран.**
  Если `startAt` ещё в будущем относительно текущего момента — статус обязан
  быть `upcoming` (или `isLive: true` если матч идёт прямо сейчас). Счёт
  выдуманного / прогнозируемого / "ожидаемого" матча — категорически нельзя.
  `completed` ставится ТОЛЬКО когда финальный счёт уже опубликован на источнике.
- Не выдумывай счёт и даты — бери только с источника. Если не уверен — оставь как было.
- Не меняй id уже существующих матчей.
- Игроков (`playerId`/`opponentId`) бери строго из `data/players.json`. Если соперника
  нет в списке игроков — `opponentId` можно оставить как есть или "TBD"; матч
  НЕ удаляй, если у `playerId` он валиден.
- Сохраняй JSON валидным, кодировка UTF-8, отступ 2 пробела.

## После правок
Запусти из корня репо: `python3 build_config.py --check`
Если валидация прошла — больше ничего не делай (сборку и пуш сделает скрипт по расписанию).
Если есть ошибки — исправь их в шардах и повтори проверку.
